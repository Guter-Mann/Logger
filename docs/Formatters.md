
# DefaultFormatter

Класс наследуемый от `logging.Formatter`, предназначен для форматировании
сообщений и внедрению кастомных данных.

## Параметры

- **fmt**: `str | None = None` - Формат сообщения по которому будут
подставлятся данные;
- **datefmt**: `str | None = None` - Формат отображения даты и времени;
- **style**: `Literal['%', '{', '$'] = '{'` - Стиль парсинга формата;
- **use_colors**: `bool = True` - Использовать цвета в выводе в терминале;
- **base_path**: `Path | None = None` - Базовый путь. Нужен для отображения
`relativePath`;
- **project_name**: `str = ''` - Имя проекта. Нужен для отображения `projectName`.

## Методы

- formatMessage(self, record: logging.LogRecord) - Основной метод вызываемый
библиотекой logging. Основная логика расположенна именно здесь;
- _safe_relative(self, pathname: str | None) - Вспомогательный метод
предназначенный для безопасного отображения пути в сообщении.

## Кастомные ключи

- **levelprefix** - Отображение уровня сообщения с цветом(при условии что
используеться консоль) и с отступом для сохранения уровня;
- **relativePath** - Относительный путь к файлу проекта;
- **projectName** - Имя проекта;
- **message** - Обновленное сообщение об ошибке с защитой чувствительной
информации;
- **name** - Имя логгера с цветом;
- **pid** - PID текущего процесса.
